Downloading the UKIRT Observing Tool

Existing users note: The current version is 20031224. If you are submitting observations for 04A you need to get this, or a newer, version of the OT. You may also need a newer version of Java (JRE).

The UKIRT Observing tool is a Java application and has been tested on Linux, Solaris, Windows 2000 and Mac OS X to various degrees - our lead platform is Linux. It is the same package as the JCMT-OT so if you have the recent version of that you are ready to go.

Go to the section describing your operating system.

  • Linux or Solaris
  • Windows 2000
  • MacOS X

Linux or Solaris users

You need three things to run the UKIRT-OT:
  1. The Java Runtime environment (JRE) version 1.4 or newer
  2. The Java Advanced Imaging extension (JAI) version 1.1.1 or newer
  3. The UKIRT-OT itself
If you are running java off the Starlink summer 2003 CD you already have the first two, so skip to step 3 (UKIRT-OT).

JRE

To check which version you have installed on your unix machine, type 
% java -version

If the answer is 1.4 you know you are ok as far as JRE is concerned. Skip to the next step (JAI). If you get an older version or "command not found" you need to install JRE and JAI. To simplify the hassle of installing both the JRE and JAI we distribute tar files containing the combined JRE and JAI packages. Versions are available for both Linux and Solaris from the OT FTP area.

This distribution can be installed simply by gunzipping and untarring it. On Solaris:

% gunzip -c jdk1_4_sun4_Solaris.tar.gz | tar xvf -

or on Linux:

% tar zxvf jdk1_4_ix86_Linux.tar.gz

This will create a directory jdk in the current directory. This is all that is required to install it. You can now skip to step 3 (UKIRT-OT)

JAI

If you have determined you have java 1.4 installed and want to check if you have JAI, you have to look for a file called jai_core.jar. This file will be in the java installation directory under the subdirectory lib/ext Here's an example of how to do it: 
% which java
/star/java/jre-1.4.1_02/bin/java
This tells us that the java installation directory is /star/java/jre-1.4.1_02/. So we would expect JAI to be in: 
 ls /star/java/jre-1.4.1_02/lib/ext/jai_core.jar
/star/java/jre-1.4.1_02/lib/ext/jai_core.jar
which it is in this example..

If it is not, you can get it directly from Sun's JAI site

UKIRT-OT

The most recent version of the OT is always available from the UKIRT-OT FTP area as ot_latest.tar.gz. The first step is simply to download it and untar the distribution [it will take up 17MB of disk space, but this distribution also includes the UKIRT-OT]. It can be untarred using:

% gunzip -c ot_latest.tar.gz | tar xvf -

This will create a directory called ompot. The installation is almost complete. If you have a java executable in your path you should now be able to run up the OT: 

% ompot/bin/ukirtot

or add the directory ompot/bin to your PATH environment variable and then type:

% ukirtot

If the location of your Java executable is not present in the path then you will need to edit the startup script so that it knows which Java runtime environment to use (this is also useful if you want to force a site wide version). You do this by editing the file ompot/bin/ot and changing the line that says:

set JAVA = `which java`

to (for example)

set JAVA = /star/java/jre/bin/java

if you are going to use the Starlink JRE. Note that the value used includes the full path to the Java executable, not a directory specification.

The OT should now be runnable as described above.

Windows 2000 users

You need three things to run the UKIRT-OT:
  1. The Java Runtime environment (JRE) version 1.4 or newer
  2. The Java Advanced Imaging extension (JAI) version 1.1.1 or newer
  3. The UKIRT-OT itself

JRE

If you have Java installed already you can check the version number by doing Start->Run->Command and then typing java -version at the MS-DOS prompt. If it is 1.4 or newer you are done. If not you can get an appropriate version from our FTP site by retrieving j2re-1_4_1_06-windows-i586.exe .

JAI

If you have not installed the UKIRT-OT before chances are you won't have JAI. You can get an appropriate version from our FTP site by retrieving jai-1_1_2-lib-windows-i586-jre.exe .

UKIRT-OT

If you are running the OT on windows the distribution is available in both .zip and .tar.gz format, you can just uncompress it using, for example, Winzip. Once it has been uncompressed go into the directory ompot\bin\ and click on the 

ukirtot
batch icon. If you have a correctly installed JRE the OT will run without any further ado.

You can get a desktop shortcut as follows: go into the directory ompot\bin\, right click on the ukirtot batch icon, and select "copy" from the resulting menu; then right-click on the desktop and select "paste shortcut". optionally rename the shortcut by right-clicking on it and selecting "rename".

MacOS X

The most recent version of the OT is always available from the UKIRT-OT FTP area as ukirt_ot_latest.sit. The first step is simply to download it and expand it (upon downloading the expansion will probably be done for you). The OT application is named UKIRT-OT and can be started by double-clicking the icon. You may copy this application to any location you desire, and it is not necessary to download and install the JAI as these libraries are included in the application package.

Note for OS X 10.3 (Panther) users: On upgrading from 10.2 to 10.3, some users have found that their Java installation is incomplete. This is evidenced by Java applications hanging, and the error message "HotSpot not at correct virtual address" appearing in the Console. If this is happening to you when attempting to run the OT, follow the instructions listed in Technical Note TN2099.
