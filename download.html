Downloading the UKIRT Observing Tool

Existing users note: The current version is 20030701. If you are submitting observations for 03B you need to get this, or a newer, version.

The UKIRT Observing tool (OT) can be downloaded from the Joint Astronomy Centre FTP server at ftp://ftp.jach.hawaii.edu/pub/ukirt/ukirtot. It is distributed in two parts:

  • A Java Runtime Environment (JRE)
  • The Observing Tool

The UKIRT Observing tool is a Java application and has been tested on Linux, Solaris and Windows 2000.

Installing the Java Runtime Environment

The OT requires the Java Runtime Environment v1.3.1 (or newer) and the Java Advanced Imaging extension v1.1.1. They must be installed on your system before attempting to run the OT. There are three ways of obtaining these packages if you don't already have them installed:

From Sun

The primary source of Java is from java.sun.com. This site hosts downloads for both JRE1.3.1 (or JRE1.4) and JAI1.1.1. Simply download it and follow the supplied instructions for installation. This is the recommended method for obtaining Windows version of Java (JRE and JAI).

JAC FTP

To simplify the difficulties of installing both the JRE and JAI we distribute tar files containing the combined JRE and JAI packages. Versions are available for both Linux and Solaris (Windows versions must be obtained from Sun) from the OT FTP area.

This distribution can be installed simply by gunzipping and untarring it. On Solaris:

% gunzip -c jdk1_3_sun4_Solaris.tar.gz | tar xvf -

or on Linux:

% tar zxvf jdk1_3_ix86_Linux.tar.gz

This will create a directory jdk in the current directory. This is all that is required to install it.

From Starlink

If you are running an Autumn 2002 Starlink release or newer you already have a JRE installed that is suitable for running the OT. It will be in your path after a Starlink login. Alternatively you can download the Starlink JRE package (which includes JAI) from the Starlink software store.

IMPORTANT! If you are running a Spring 2002 Starlink distribution you already have JRE1.3.1 installed in /star/java/jre-1.3.1_02/bin/java (this will vary depending on where you installed the Starlink software) but the Spring 2002 Starlink distribution does not include the JAI extension. You can either download that separately from Sun (see the previous note) or run the OT without the JAI (the OT will run fine but you will not be able to use the position editor).

We have tested the OT with JRE1.4.1 and it seems to work fine.

Installing the OT

The most recent version of the OT is always available from the UKIRT-OT FTP area as ot_latest.tar.gz. The first step is simply to download it and untar the distribution [it will take up 17MB of disk space, but this distribution also includes the UKIRT-OT]. It can be untarred using:

% gunzip -c ot_latest.tar.gz | tar xvf -

This will create a directory called ompot. The installation is almost complete. If you have a java executable in your path you should now be able to run up the OT: 

% ompot/bin/ukirtot

or add the directory ompot/bin to your PATH environment variable and then type:

% ukirtot

If the location of your Java executable is not present in the path then you will need to edit the startup script so that it knows which Java runtime environment to use (this is also useful if you want to force a site wide version). You do this by editing the file ompot/bin/ot and changing the line that says:

set JAVA = `which java`

to (for example)

set JAVA = /star/java/jre/bin/java

if you are going to use the Starlink JRE. Note that the value used includes the full path to the Java executable, not a directory specification.

The OT should now be runnable as described above.

You are now ready for the OT Primer.

Windows users

If you are running the OT on windows, you can just uncompress it using, for example, Winzip. Once it has been uncompressed go into the directory ompot\bin\ and click on the 

ukirtot
batch icon. If you have a correctly installed JRE the OT will run without any further ado.

You can get a desktop shortcut as follows: go into the directory ompot\bin\, right click on the ukirtot batch icon, and select "copy" from the resulting menu; then right-click on the desktop and select "paste shortcut". optionally rename the shortcut by right-clicking on it and selecting "rename".
